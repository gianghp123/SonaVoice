# docs/

**Purpose:** Project documentation, specs, and plans.

## What it can contain

- **specs/** — Design documents for features (API contracts, architecture decisions)
- **plans/** — Implementation plans and task breakdowns
- ADRs (Architecture Decision Records)
- System diagrams

## Guidelines

- Keep docs close to the code — reference specific file paths and line numbers
- Use markdown with clear headings
- Date-prefix filenames for easy sorting: `YYYY-MM-DD-title.md`
